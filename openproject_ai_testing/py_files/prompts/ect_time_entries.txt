You are a software tester performing Equivalence Class Testing (ECT) on the OpenProject REST API.

Target endpoint: Time Entries
Base URL: {BASE_URL}
Authentication: Basic auth with username "apikey" and password "{API_KEY}"

The Time Entries API works as follows:
- GET    /api/v3/time_entries        -> list all time entries
- POST   /api/v3/time_entries        -> create a time entry
- GET    /api/v3/time_entries/{id}   -> get a time entry
- PATCH  /api/v3/time_entries/{id}   -> update a time entry
- DELETE /api/v3/time_entries/{id}   -> delete a time entry

When creating a time entry, the request body must be JSON with these fields:
- "hours": string in ISO 8601 duration format (required, e.g. "PT1H" for 1 hour, "PT30M" for 30 minutes)
- "spentOn": string in YYYY-MM-DD date format (required)
- "_links": object containing:
    - "workPackage": object with "href" set to "/api/v3/work_packages/{id}" (required)
    - "activity": object with "href" set to "/api/v3/time_entries/activities/1" (optional — OpenProject applies a default activity if omitted)

Use work package ID {WORK_PACKAGE_ID} for all tests. If no work package ID is available in the environment, use work package ID 1.

Important known behaviors:
- "PT0S" (zero hours) should be rejected with 422
- Negative durations like "PT-1H" should be rejected with 422
- Missing activity link is accepted — OpenProject silently applies a default activity

Equivalence Class Testing means you divide inputs into classes where all values in a class are expected to behave the same way, then test one representative value from each class:

For the "hours" field:
- Valid class 1: a positive duration in hours e.g. "PT2H" (should succeed with 201)
- Valid class 2: a positive duration in minutes e.g. "PT45M" (should succeed with 201)
- Valid class 3: a combined duration e.g. "PT1H30M" (should succeed with 201)
- Invalid class 1: missing hours field entirely (should fail with 422)
- Invalid class 2: zero duration "PT0S" (should fail with 422)
- Invalid class 3: negative duration "PT-1H" (should fail with 422)
- Invalid class 4: invalid format string e.g. "2 hours" (should fail with 422)

For the "spentOn" field:
- Valid class: a valid date string in YYYY-MM-DD format (should succeed with 201)
- Invalid class 1: missing spentOn field entirely (should fail with 422)
- Invalid class 2: invalid date format e.g. "28-04-2025" (should fail with 422)
- Invalid class 3: non-date string e.g. "yesterday" (should fail with 422)

For the "_links.workPackage" field:
- Valid class: valid href pointing to an existing work package (should succeed with 201)
- Invalid class 1: missing workPackage link entirely (should fail with 422)
- Invalid class 2: href pointing to a non-existent work package e.g. "/api/v3/work_packages/99999" (should fail with 422)

For the "_links.activity" field:
- Valid class: valid href pointing to an existing activity e.g. "/api/v3/time_entries/activities/1" (should succeed with 201)
- Omitted class: activity link omitted entirely (should succeed with 201 — default applied)
