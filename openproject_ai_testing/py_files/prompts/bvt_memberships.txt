Target endpoint: Memberships
Base URL: {BASE_URL}
Authentication: Basic auth with username "apikey" and password "{API_KEY}"

The Memberships API works as follows:
- GET    /api/v3/memberships          -> list all memberships
- POST   /api/v3/memberships          -> create a membership
- GET    /api/v3/memberships/{id}     -> get a membership
- PATCH  /api/v3/memberships/{id}     -> update a membership
- DELETE /api/v3/memberships/{id}     -> delete a membership

When creating a membership, the request body must be JSON with these fields:
- "_links.project": object with "href" pointing to a project e.g. {"href": "/api/v3/projects/1"}
- "_links.principal": object with "href" pointing to a user e.g. {"href": "/api/v3/users/4"}
- "_links.roles": array of role objects e.g. [{"href": "/api/v3/roles/3"}]

Real data available in this system:
- Projects: 1=Demo project, 2=Scrum project, 3=Test project
- Users: 4=Admin user (already a member of projects 1 and 2)
- Roles: 1=Non member, 2=Anonymous, 3=Member, 4=Reader, 5=Project admin, 6=Work package editor, 7=Work package commenter, 8=Work package viewer

Important known behaviors:
- User deletion via the API returns 403. To test with fresh users, create them first via POST /api/v3/users then use their IDs.
- Deleting a membership via DELETE /api/v3/memberships/{id} returns 204 on success.
- Creating a membership for a user already in a project returns 422.

Boundary Value Testing means you test at the exact boundaries of valid input ranges:

For roles array:
- Minimum valid: exactly 1 role (should succeed with 201)
- Multiple roles: 2 roles at once (should succeed with 201)
- Empty roles array: [] (should fail with 422)
- Missing roles entirely (should fail with 422)

For project link:
- Valid project href (should succeed with 201)
- Non-existent project href e.g. /api/v3/projects/99999 (should fail with 422)
- Missing project link (should fail with 422)

For principal link:
- Valid user href (should succeed with 201)
- Non-existent user href e.g. /api/v3/users/99999 (should fail with 422)
- Missing principal link (should fail with 422)
