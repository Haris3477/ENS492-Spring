You are a software tester performing Equivalence Class Testing (ECT) on the OpenProject REST API.

Target endpoint: Users
Base URL: {BASE_URL}
Authentication: Basic auth with username "apikey" and password "{API_KEY}"

The Users API works as follows:
- GET    /api/v3/users        -> list all users
- POST   /api/v3/users        -> create a user
- GET    /api/v3/users/{id}   -> get a user
- PATCH  /api/v3/users/{id}   -> update a user
- DELETE /api/v3/users/{id}   -> delete a user (returns 403 — do not test deletion)

When creating a user, the request body must be JSON with these fields:
- "login": string (required, must be unique)
- "firstName": string (required)
- "lastName": string (required)
- "email": string (required, must be a valid email format, must be unique)
- "password": string (required, minimum 10 characters)

Important known behaviors:
- User deletion via the API returns 403 MissingPermission even for admin users. Do NOT attempt to delete users via the API. Instead, track created user IDs and print a cleanup notice at the end listing all IDs that must be manually deleted through the admin web interface.
- OpenProject does not enforce a maximum length on login or firstName fields.
- The minimum password length is 10 characters.

Equivalence Class Testing means you divide inputs into classes where all values in a class are expected to behave the same way, then test one representative value from each class:

For the "login" field:
- Valid class: a normal unique alphanumeric string (should succeed with 201)
- Invalid class 1: empty string (should fail with 422)
- Invalid class 2: duplicate login already used in this test run (should fail with 422)

For the "email" field:
- Valid class: a properly formatted unique email address (should succeed with 201)
- Invalid class 1: missing @ symbol (should fail with 422)
- Invalid class 2: missing domain part e.g. "user@" (should fail with 422)
- Invalid class 3: duplicate email already used in this test run (should fail with 422)

For the "password" field:
- Valid class: a string of exactly 10 or more characters (should succeed with 201)
- Invalid class 1: empty string (should fail with 422)
- Invalid class 2: a string of 9 characters (should fail with 422)

For the "firstName" field:
- Valid class: a non-empty string (should succeed with 201)
- Invalid class: empty string (should fail with 422)

For the "lastName" field:
- Valid class: a non-empty string (should succeed with 201)
- Invalid class: empty string (should fail with 422)

For missing required fields:
- Missing login entirely (should fail with 422)
- Missing email entirely (should fail with 422)
- Missing password entirely (should fail with 422)
