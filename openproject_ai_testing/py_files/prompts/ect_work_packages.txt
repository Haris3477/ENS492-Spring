Target endpoint: Work Packages

The Work Packages API works as follows:
- GET    /api/v3/work_packages                    -> list all work packages
- POST   /api/v3/projects/{id}/work_packages      -> create a work package in a project
- GET    /api/v3/work_packages/{id}               -> get a work package
- PATCH  /api/v3/work_packages/{id}               -> update a work package
- DELETE /api/v3/work_packages/{id}               -> delete a work package

When creating a work package, the request body must be JSON with these fields:
- "subject": string (required, must not be empty, maximum 255 characters)
- "description": object with "raw" string field (optional)
- "_links": object containing:
    - "type": object with "href" set to "/api/v3/types/1" (required)

The project ID to use is {PROJECT_ID} and the project identifier is {PROJECT_IDENTIFIER}.

Equivalence Class Testing means you divide inputs into classes where all values in a class are expected to behave the same way, then test one representative value from each class:

For the "subject" field:

- Invalid class 1: empty string (should fail with 422)
- Invalid class 2: string longer than 255 characters (should fail with 422)

- Valid class 1: description object with non-empty raw string (should succeed with 201)
