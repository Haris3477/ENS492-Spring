=== WORK PACKAGE FIELDS ===

When creating or updating a work package, the request body is JSON with these fields:

- "subject": string (required, must not be empty, maximum 255 characters)
- "description": object with "raw" string field (optional)
- "startDate": string in "YYYY-MM-DD" format (optional)
- "dueDate": string in "YYYY-MM-DD" format (optional, must not be before startDate)
- "estimatedTime": string in ISO 8601 duration format e.g. "PT2H" for 2 hours, "PT30M" for 30 minutes (optional)
- "percentageDone": integer between 0 and 100 inclusive (optional)
- "_links": object containing:
    - "type": object with "href" pointing to a valid type (optional, defaults to Task)
    - "status": object with "href" pointing to a valid status (optional, defaults to New)
    - "priority": object with "href" pointing to a valid priority (optional, defaults to Normal)
    - "assignee": object with "href" pointing to a valid user e.g. "/api/v3/users/1" (optional)

=== EQUIVALENCE CLASS TEST CASES ===

Equivalence Class Testing means you divide inputs into classes where all values in a class behave the same way, then test one representative value from each class.

SUBJECT field:
- Valid class: normal string 1–255 characters (expect 201)
- Invalid class 1: empty string (expect 422)
- Invalid class 2: string longer than 255 characters (expect 422)
- Invalid class 3: missing subject entirely (expect 422)

TYPE field:
- Valid class 1: valid type href e.g. /api/v3/types/1 (expect 201)
- Valid class 2: valid type href for a different type e.g. /api/v3/types/7 for Bug (expect 201)
- Invalid class 1: non-existent type href e.g. /api/v3/types/99999 (expect 422)
- Valid class 3: missing _links.type entirely (expect 201, OpenProject applies default type)

STATUS field:
- Valid class 1: valid open status e.g. /api/v3/statuses/1 for New (expect 201)
- Valid class 2: valid closed status e.g. /api/v3/statuses/12 for Closed (expect 201)
- Invalid class: non-existent status href e.g. /api/v3/statuses/99999 (expect 422)

PRIORITY field:
- Valid class 1: low priority /api/v3/priorities/7 (expect 201)
- Valid class 2: high priority /api/v3/priorities/9 (expect 201)
- Invalid class: non-existent priority href e.g. /api/v3/priorities/99999 (expect 422)

DESCRIPTION field:
- Valid class 1: description with non-empty raw string (expect 201)
- Valid class 2: description with empty raw string (expect 201)
- Valid class 3: description field omitted entirely (expect 201)

DATE field:
- Valid class: startDate before dueDate (expect 201)
- Invalid class 1: dueDate before startDate (expect 422)
- Invalid class 2: invalid date format e.g. "not-a-date" (expect 422)

ESTIMATED TIME field:
- Valid class: proper ISO 8601 duration e.g. "PT2H" (expect 201)
- Invalid class: wrong format e.g. "2 hours" (expect 422)

PERCENTAGE DONE field:
- Valid class 1: value within range e.g. 50 (expect 201)
- Invalid class 1: value below range e.g. -1 (expect 422)
- Invalid class 2: value above range e.g. 101 (expect 422)
