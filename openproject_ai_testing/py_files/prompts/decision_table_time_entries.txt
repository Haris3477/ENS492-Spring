You are a software tester performing Decision Table Testing on the OpenProject REST API.

Target endpoint: Time Entries
Base URL: {BASE_URL}
Authentication: Basic auth with username "apikey" and password "{API_KEY}"

The Time Entries API works as follows:
- GET    /api/v3/time_entries        -> list all time entries
- POST   /api/v3/time_entries        -> create a time entry
- GET    /api/v3/time_entries/{id}   -> get a time entry
- PATCH  /api/v3/time_entries/{id}   -> update a time entry
- DELETE /api/v3/time_entries/{id}   -> delete a time entry

When creating a time entry, the request body must be JSON with these fields:
- "hours": string in ISO 8601 duration format (required, e.g. "PT1H" for 1 hour, "PT30M" for 30 minutes)
- "spentOn": string in YYYY-MM-DD date format (required)
- "_links": object containing:
    - "workPackage": object with "href" set to "/api/v3/work_packages/{id}" (required)
    - "activity": object with "href" set to "/api/v3/time_entries/activities/1" (optional — OpenProject applies a default activity if omitted)

Use work package ID from environment variable OP_WORK_PACKAGE_ID (default: "39").

Important known behaviors:
- "PT0S" (zero hours) is rejected with 422
- Negative durations like "PT-1H" are rejected with 422
- Missing activity link is accepted — OpenProject silently applies a default activity

Decision Table Testing means you test combinations of input conditions together and verify the outcome. Each rule in the decision table specifies the state of multiple inputs at once and the expected result.

Here is the decision table to implement. Each row is one test case combining multiple conditions:

Rule 1:  hours=valid(PT2H), spentOn=valid, workPackage=valid, activity=valid     -> expect 201
Rule 2:  hours=valid(PT30M), spentOn=valid, workPackage=valid, activity=omitted  -> expect 201 (default activity applied)
Rule 3:  hours=missing, spentOn=valid, workPackage=valid, activity=valid         -> expect 422
Rule 4:  hours=valid, spentOn=missing, workPackage=valid, activity=valid         -> expect 422
Rule 5:  hours=valid, spentOn=valid, workPackage=missing, activity=valid         -> expect 422
Rule 6:  hours=zero(PT0S), spentOn=valid, workPackage=valid, activity=valid      -> expect 422
Rule 7:  hours=negative(PT-1H), spentOn=valid, workPackage=valid, activity=valid -> expect 422
Rule 8:  hours=valid, spentOn=invalid format, workPackage=valid, activity=valid  -> expect 422
Rule 9:  hours=missing, spentOn=missing, workPackage=valid, activity=valid       -> expect 422
Rule 10: hours=missing, spentOn=valid, workPackage=missing, activity=valid       -> expect 422
Rule 11: hours=valid(PT1H30M), spentOn=valid, workPackage=valid, activity=valid  -> expect 201
Rule 12: hours=invalid string, spentOn=valid, workPackage=valid, activity=valid  -> expect 422

        if links:

        response = requests.post(f"{BASE_URL}/api/v3/time_entries", json=body, auth=auth)
        return response
